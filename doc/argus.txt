1. INTRODUCTION                                           *argus-introduction*

Argus is a Neovim plugin that displays a hierarchical outline of Go source
code symbols. It allows you to navigate your code structure and reorder
symbols by moving them up/down directly from the outline window.

Features:
- Hierarchical symbol outline for Go files
- Two view modes: flat (file order) and hierarchy (grouped by type)
- Move symbols up/down to reorganize code
- Methods grouped under their receiver types (in hierarchy view)
- Struct fields and interface methods as children
- Doc comments move with their associated symbols
- Live filtering/search
- Cursor synchronization between source and outline

==============================================================================
2. REQUIREMENTS                                           *argus-requirements*

- Neovim >= 0.9.0
- nvim-treesitter with Go parser installed

To install the Go treesitter parser:
>lua
    :TSInstall go

3. INSTALLATION                                           *argus-installation*

Using lazy.nvim: ~
>lua
    {
      "yourusername/argus.nvim",
      ft = "go",
      opts = {},
      keys = {
        { "<leader>cs", "<cmd>ArgusToggle<cr>", desc = "Toggle Code Outline" },
      },
    }
<

Using packer.nvim: ~
>lua
    use {
      "yourusername/argus.nvim",
      config = function()
        require("argus").setup()
      end,
      ft = "go",
    }
<

Local plugin (development): ~
>lua
    {
      dir = "~/path/to/argus.nvim",
      ft = "go",
      opts = {},
    }
<

>lua
    require("argus").setup({
      -- Window settings
      position = "right",      -- "left" | "right"
      width = 40,              -- window width
      auto_close = true,       -- close when source buffer closes

      -- Display settings
      icons = {
        package = "Û∞èó",
        ["function"] = "Û∞äï",
        method = "Û∞Üß",
        struct = "Û∞ôÖ",
        interface = "Û∞ú∞",
        type = "Û∞äÑ",
        const = "Û∞èø",
        var = "Û∞Ä´",
        field = "Û∞ú¢",
        collapsed = "",
        expanded = "",
      },
      show_line_numbers = true,

      -- Behavior settings
      auto_preview = false,    -- preview symbol on cursor move
      follow_cursor = true,    -- sync outline cursor with source
      view_mode = "flat",      -- "flat" | "hierarchy"

      -- Keymaps (in outline window)
      keymaps = {
        close = "q",
        jump = "<CR>",
        move_up = "K",
        move_down = "J",
        toggle_fold = "za",
        expand_all = "zR",
        collapse_all = "zM",
        filter = "/",
        clear_filter = "<Esc>",
        refresh = "R",
        toggle_view = "v",
        help = "?",
      },
    })
<

                                                        *argus-config-position*
position ~
    Window position: "left" or "right"
    Default: "right"

                                                           *argus-config-width*
width ~
    Width of the outline window in columns.
    Default: 40

                                                      *argus-config-auto_close*
auto_close ~
    Automatically close the outline when the source buffer is closed.
    Default: true

                                                           *argus-config-icons*
icons ~
    Table of icons for each symbol type. Requires a Nerd Font.
    You can use one of the built-in presets (see |argus-icon-presets|).

                                               *argus-config-show_line_numbers*
show_line_numbers ~
    Show line numbers next to symbols.
    Default: true

                                                    *argus-config-auto_preview*
auto_preview ~
    Preview symbol location when moving cursor in outline.
    Default: false

                                                   *argus-config-follow_cursor*
follow_cursor ~
    Sync outline cursor position with source cursor.
    Default: true

                                                       *argus-config-view_mode*
view_mode ~
    Initial view mode: "flat" (file order) or "hierarchy" (grouped).
    Default: "flat"

                                                         *argus-config-keymaps*
keymaps ~
    Table of keybindings for the outline window.

                                                                 *:ArgusClose*
:ArgusClose ~
    Close the outline window.

:ArgusToggle ~
    Toggle the outline window.

                                                               *:ArgusRefresh*
:ArgusRefresh ~
    Refresh the outline to reflect changes in the source.

6. KEYBINDINGS                                             *argus-keybindings*

These keybindings are active in the outline window:

    Key         Action ~
    -----------------------------------------------------------------------
    q           Close the outline window
    <CR>        Jump to symbol in source buffer
    K           Move symbol up in source code
    J           Move symbol down in source code
    za          Toggle fold (expand/collapse)
    zR          Expand all folds
    zM          Collapse all folds
    /           Open filter prompt
    <Esc>       Clear filter
    R           Refresh outline
    v           Toggle view mode (flat/hierarchy)
    ?           Show help popup

                                                             *argus-move-up*
Moving Symbols Up (K) ~
    Moves the symbol at cursor position up, swapping it with the previous
    sibling symbol. Doc comments are moved along with the symbol.

                                                           *argus-move-down*
Moving Symbols Down (J) ~
    Moves the symbol at cursor position down, swapping it with the next
    sibling symbol. Doc comments are moved along with the symbol.

                                                              *argus-filter*
Filtering (/) ~
    Opens a live filter prompt. As you type, the outline updates to show
    only symbols matching the pattern. Press <CR> to confirm or <Esc> to
    cancel.

>lua
    vim.api.nvim_set_hl(0, "ArgusFunction", { fg = "#7aa2f7", bold = true })

11. API                                                           *argus-api*

The plugin exposes a Lua API for advanced usage:

                                                              *argus.setup()*
argus.setup({opts}) ~
    Initialize the plugin with configuration options.

                                                               *argus.open()*
argus.open([bufnr]) ~
    Open the outline for the given buffer (or current buffer).

                                                              *argus.close()*
argus.close() ~
    Close the outline window.

                                                             *argus.toggle()*
argus.toggle() ~
    Toggle the outline window.

                                                            *argus.refresh()*
argus.refresh() ~
    Refresh the outline to reflect source changes.

                                                            *argus.is_open()*
argus.is_open() ~
    Returns true if the outline window is open.

Submodules: ~

    `require("argus").window`     Window management
    `require("argus").render`     Rendering logic
    `require("argus").parser`     Treesitter parsing
    `require("argus").actions`    User actions
    `require("argus").filter`     Filter functionality
    `require("argus").config`     Configuration

Example: ~
>lua
    local argus = require("argus")

    -- Open outline with a keymap
    vim.keymap.set("n", "<leader>cs", argus.toggle, { desc = "Toggle Outline" })

    -- Check if open before doing something
    if argus.is_open() then
      argus.refresh()
    end
<
